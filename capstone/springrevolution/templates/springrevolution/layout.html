Users’ passwords, are stored as hashed characters (random characters) in the database using hash function which is one way (i.e. Even we cannot see the password users have typed because your passwords, hashed characters, cannot be converted into plain text again).

While we work to protect the confidentiality of users' indentity related information, email addresses in specific, we cannot guarantee that unauthorized third parties will not be able to defeat our security measures.

Data of this website, specifically links of donations and messages, is compiled with the help of users. Thus, users’ contribution to the website is highly appreciated, and it can be done via the forms under their respective tabs. It’s worth noting that the data submitted via the forms in question will be sent to us first, from which whether or not the submitted data should be approved will be decided. This process of sanitising the incoming data means that the submitted data will not be directly added to the website.

  About Donation Links
Donation links are arranged by the following order which we presume is the most optimal for efficiency:
  1. Sources (first priority)
    1. Website
    2. Other Social Platform
  2. Date (second priority)
    1. Null
    2. Latest
    3. Oldest
  3. Name (third priority)
       - alphabetical order
ALERT: There are two things users should bear in mind about donation links.
  1. Though the submission for adding links is immediately denied if the link is already in the database, there is still the possibility of location duplications due to some cases where different URLs lead to the same location.
  2. Despite each and every link submitted being mindfully checked before being added into the database, it cannot be completely guaranteed that the charities or individuals led by the links are trustworthy or still accepting donations. Therefore, users should evaluate their trustworthiness only after visiting the links and learning about them. We will work to progressively mitigate this potential inconvenience as the website scales up.
About Messaging Services
By subscribing the service, users will receive an email in their Gmail inbox once in a specific time interval they have set. As a sample, an email will be sent as soon as users have subscribed so that they can decide their subscription status. Most messages in our emails are written by other fellow users, and because the message for each email is chosen randomly from the database, it is possible that users receive the same email they've got in earlier days. Nevertheless, the more messages left by users, the more this potential inconvenience can be mitigated.
Security & Privacy Conditions
We do prioritize the security of our users especially due to current situations in Myanmar. For each user, we store six information, in which user’s email address is the only identity related information. Other information includes subscription related data, nickname, and password. Of course, all data of users will be kept highly confidential.

When it comes to users’ passwords, they will be stored as hashed characters (random characters) in the database using hash function which is one way (i.e. Even we cannot see the password users have typed because your passwords, hashed characters, cannot be converted into plain text again).

While we work to protect the security of users' indentity related information, we cannot guarantee that unauthorized third parties will not be able to defeat our security measures. Please notify us immediately of any compromise or unauthorized use of your account by reporting us from here.
